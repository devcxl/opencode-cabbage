## 目标

为 Agent 增加正式的 `permission` 字段声明和静态 Lint 校验，并通过 `npm pack` smoke test 确认 TDD skill 随包发布。

## 实现要点

1. **Agent `permission` 字段解析** (`src/plugin/agents.ts`)
   - `AgentEntry` 增加 `permission` 字段解析
   - `capabilities` 仅用于 Lint 和文档校验
   - 将解析后的 permission 传递给 OpenCode config

2. **Agent frontmatter 更新**
   - `backend.md` / `frontend.md`：声明 `permission: {bash: "npm test|git push|...", write: ".worktree/", edit: "src/,test/,assets/"}` — 不含 GitHub 写操作
   - `reviewer.md`：声明 `permission: {bash: "gh pr view|diff|checks", write: deny, edit: deny}`

3. **prompt-lint 新增规则** (`src/plugin/prompt-lint.ts`)
   - 检测 agent frontmatter 是否缺少 `permission` 字段
   - 检测 Worker 中是否有 `gh pr create|merge` 等权限越界
   - 检测 `capabilities` 是否与 `permission` 一致
   - 检测 Reviewer 是否声明了写权限

4. **npm pack smoke test**
   - 在 `package.json` 添加 `test:pack` 脚本
   - 验证打包后 tarball 包含 `assets/skills/flow-tdd/SKILL.md`

## 验收标准

- [ ] 4 个 agent prompts 均包含合法 `permission` 字段
- [ ] prompt-lint 新规则通过 CI
- [ ] `npm pack` tarball 包含 `flow-tdd/SKILL.md`
- [ ] Agent 测试覆盖 permission 解析
