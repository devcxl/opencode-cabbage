## 目标

实现 TDD 模式判定纯函数，覆盖 Spec Section 4.3 完整真值表。

## 实现要点

1. **Evaluator 函数** (`src/flowrun/evaluator.ts`)
   ```typescript
   evaluateTddCompliance(policy: TddPolicy, evidence: TddEvidence): EvaluationResult
   ```
   - 输入：冻结的 `TddPolicy` + `TddEvidence`
   - 输出：`EvaluationResult`（pass/fail/waived + warnings）
   - 不访问文件系统、网络或任何运行时状态

2. **核心规则**
   - `strict`：至少一个 TDD criterion；每个 TDD criterion 被 cycle 覆盖；cycle 状态为 `pass`
   - `relaxed`：仅要求 regression 为 `pass`，不检查 cycle
   - `bypass`：要求全部 alternative validation 为 `pass`，返回 `waived`
   - `verification: regression` criterion 由 final regression 覆盖
   - `verification: manual` 仅允许 bypass，需要审批引用
   - Advisory 模式：缺 evidence 不阻断，仅产生 warning
   - Runtime 模式：缺 evidence 阻断

3. **测试覆盖**
   - strict + advisory → pass with warnings
   - strict + runtime + 所有 criterion 有 cycle → pass
   - strict + runtime + 缺一个 criterion cycle → fail
   - relaxed + advisory → pass with warnings
   - relaxed + runtime + regression pass → pass
   - bypass + runtime + 替代验证完成 → waived
   - bypass + runtime + 替代验证缺失 → fail
