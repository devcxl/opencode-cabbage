# 代码气味基线

参考 Fowler《重构》第 3 章。这些气味仅在审查 diff 中出现时报告；跳过工具已强制执行的部分。

## 气味清单

| 气味 | 表现 | 建议修复 |
|---|---|---|
| 神秘命名 | 变量/函数名无法表达意图 | 重命名，让名字自解释 |
| 重复代码 | 相同逻辑出现在多处 | 提取函数/类 |
| 特性依恋 | 一个方法过度使用其他类的数据 | 将逻辑移到数据所属的类 |
| 数据泥团 | 相同的数据项组合反复出现 | 提取为值对象 |
| 基本类型偏执 | 用 string/number 表示领域概念 | 创建值类型/枚举 |
| 霰弹式修改 | 一个改动需要改多处 | 用 Move Method/Field 集中职责 |
| 发散式变更 | 一个类因不同原因频繁修改 | 拆分类，分离关注点 |
| 臆测通用性 | 为不存在的未来需求做的抽象 | YAGNI — 删除 |
| 消息链 | a.b().c().d() 式的长链调用 | 隐藏委托 |
| 中间人 | 类的主要工作是委托给其他类 | 删除中间层 |
