## 目标

实现 Goal-FlowRun 终态绑定和 `flow_run finalize` 幂等流程，同步所有文档。

## 实现要点

1. **Goal 终态绑定** (`src/plugin/goal.ts`)
   - `goal.complete` 存在 FlowRunRef 时：
     - 读取精确 FlowRun → 检查 status 为 `completed`
     - 非 `completed` → 返回 `FLOW_RUN_INCOMPLETE`
     - 存在引用但缺失 FlowRun → 返回 `FLOW_RUN_NOT_FOUND`
   - Goal 写入失败可重试，不重复合并或修改 Stage

2. **`flow_run finalize`** (`src/flowrun/index.ts`)
   - `finalizeFlowRun(parentIssueNumber)`：
     - 确认所有 Task 为 `merged`
     - code Stage: 确认所有 Task TDD/实现已合并 → `pass`
     - test Stage: 聚合各 PR 的 regression/CI/coverage → `pass`
     - review Stage: 聚合 reviewer + acceptance → `pass`
     - merge Stage: 确认所有 mergeResult 通过 → `pass`
   - 幂等：重复调用不影响已完成的 Stage
   - 全局 `canMerge()` 在此处使用（非 Task 级合并）

3. **文档同步**
   - `docs/guides/architecture.md`：新增 TDD 三层约束模型架构描述
   - `docs/guides/configuration.md`：新增 TDD policy/enforcement/mode 配置说明
   - `docs/guides/usage.md`：新增 TDD 工作流（Advisory→Runtime→Repository Quality）使用指南
   - `assets/prompts/bootstrap.md`：更新 TDD enforcement 默认配置说明
