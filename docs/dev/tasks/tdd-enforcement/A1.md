## 目标

建立 Phase A Advisory 阶段的 TDD 协议。新增 `flow-tdd` skill 作为 TDD Prompt 协议唯一来源，并更新所有相关 skills 和 agent prompts 以引用统一协议。

## 实现要点

1. **新增 `assets/skills/flow-tdd/SKILL.md`**
   - 遵循 ADR 0006 八段 Contract 格式
   - Advisory Procedure：描述 RED→GREEN 手动流程、self-reported 标记
   - Runtime Procedure（占位）：注释标注"Phase B 启用"，供后续替换
   - 协议：`cycle-start → red → green`，`abandon-cycle` 规则，`final-regression` + `final-verification` 要求

2. **更新 `flow-code/SKILL.md`**
   - 新增 Section：TDD Advisory Protocol — 引用 `flow-tdd`
   - commit/push 前增加 TDD 检查提醒

3. **更新 `flow-tasks/SKILL.md`**
   - Task 生成时输出结构化 `acceptance_criteria`（含 verification type）
   - 输出 `test_commands`、`verify_commands`、`tdd` 配置块
   - 模式默认 `strict`

4. **更新 `flow-review/SKILL.md`**
   - 检查 criterion coverage 和风险
   - 明确不检查 commit order

5. **更新 `dev-lifecycle.md`**
   - 调用 broker tools，不直接执行 `gh pr create/merge`
   - 引用 flow-tdd protocol

6. **更新 Worker agents（backend/frontend）**
   - 加载 `flow-tdd` skill
   - 不复制状态机逻辑

## 验收标准

- [ ] `flow-tdd/SKILL.md` 存在并符合 Contract 格式
